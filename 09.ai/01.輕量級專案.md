# 輕量級專案：AI 開發快速指南

本文件詳述適合小型專案、個人實驗或短期工具腳本的開發架構。比起專業級的多重角色分權，輕量級專案強調 **「最小化行政管理，最大化執行速度」**。

## 💡 治理建議：雙軌制選擇

並非所有修改都要跑正式流程。您可以根據修改的複雜度選擇模式：

| 修改類型 | 建議模式 | 示例 |
| :--- | :--- | :--- |
| **微調 (Minor Tweak)** | **對話式修改** | 改錯字、補一行註釋、調整標題。直接在對話中處理。 |
| **任務 (Task)** | **輕量級流程 (TASK.md)** | 新增一個 FAQ 章節、更新安裝手冊。在 `TASK.md` 紀錄後直接執行。 |
| **工程 (Project)** | **專業級 (Conductor)** | 整個專案的文件風格轉換、整合多份手冊成一套 Wiki。 |

---

## 🏃 核心理念：極簡治理

輕量級專案將專業級的「五大角色」濃縮為兩大核心支柱：
1.  **動態執行 (Total Manager)**：由 `TASK.md` 負責。
2.  **靜態約束 (Simplified Law)**：由單一的 `GUIDELINES.md` 負責。

### 專業級 vs. 輕量級 檔案差異

| 項目 | 專業級 (Professional) | 輕量級 (Lightweight) | 作用說明 |
| :--- | :--- | :--- | :--- |
| **動態進度** | `.conductor/`, `plan.md`, `TASK.md` | `TASK.md` (單一層級) | 追蹤 AI 的當前任務與完成狀態。 |
| **技術法規** | `GUIDELINES.md`, `ARCHITECT.md`, `ADR.md` | `GUIDELINES.md` | 定義代碼規範、架構決策與互動準則。 |
| **環境約束** | `.env`, `.gitignore`, `.eslintrc` | `.env`, `.gitignore` | 設定開發環境變數與檔案過濾規則。 |
| **特殊技能** | `CUSTOM_SKILL.md` | (通常省略，併入 GUIDELINES) | 擴充 AI 的工具集或特定業務邏輯 SOP。 |
| **驗證測試** | `tests/`, `test_report.md` | (通常省略) | 確保功能正確性與回歸測試。 |

## 📂 輕量級檔案詳細說明 (以 ASP.NET MVC 校務系統為例)

以下說明如何在輕量級架構下使用這些檔案，並提供實際的程式片段：

### 1. TASK.md (動態進度檔案)
當您交辦任務給 AI 時，AI 必須根據此檔案更新進度。它就像一個**看板**。
*   **範例內容**：
    ```markdown
    # 任務：開發「學生成績查詢」模組
    - [x] 在 `GradeController` 建立 Index 頁面
    - [/] 實作 SQL 檢索邏輯：目前卡在多年度加權計算
    - [ ] 撰寫 Excel 匯出功能
    ```

### 2. GUIDELINES.md (靜態規範檔案)
這是您對系統的**法律定義**。AI 產生的每一行代碼都必須合規。
*   **範例內容 (校務開發專屬)**：
    ```markdown
    ## 🗄️ SQL 規範
    *   欄位名稱：全大寫 (如 `STU_ID`)，並對齊資料庫架構。
    *   關鍵字：小寫 (如 `select`, `from`)。
    *   關聯：優先使用 `left join`。
    ## ✒️ 命名規範
    *   C# 方法：PascalCase (如 `CalculateWeightedGpa`)。
    *   JS 變數：camelCase (如 `studentScoreData`)。
    ```

### 3. .env (環境設定檔)
用來存放**機密資訊**或**環境參數**，不進入版控系統。
*   **範例內容**：
    ```bash
    DB_CONNECTION="Server=prod-server;Database=SchoolDB;User Id=sa;Password=SafePassword123;"
    ENVIRONMENT="Production"
    ```

### 4. CUSTOM_SKILL.md (業務經驗/特殊技能)
如果有些邏輯很難寫在法規裡，或屬於特定領域的「Know-how」，就寫在**技能說明**中。這能防止 AI 寫出雖然代碼正確但「業務邏輯錯誤」的程式。

*   **詳細業務範例 (校務專家)**：
    > **1. 學分計算 (GPA)**：
    > *   必修課加權為 1.5，選修課為 1.0。
    > *   總學分低於 128 不得計算畢業門檻。
    >
    > **2. 學期成績排定**：
    > *   平時成績 (30%) + 期中考 (30%) + 期末考 (40%)。
    > *   若學生缺課次數超過學期總時數 1/3，該科直接以「扣考 (0分)」處理，不進入加權計算。
    >
    > **3. 選課擋修規則 (Prerequisites)**：
    > *   修習「進階程式設計」前，必須先取得「程式設計(一)」的學分且成績高於 70 分。
    > *   如果是轉學生，則改審核 `CREDIT_TRANSFER` 欄位是否包含 `PROG_BASIC` 標籤。

## 🛠️ 新專案：如何快速初始化

當您要開啟一個新腳本或小功能時，只需三步：

### SOP 流程：
1.  **建立 `GUIDELINES.md`**：定義最基本的代碼風格與技術棧。
2.  **建立 `TASK.md`**：寫下第一階段要做的 3-5 個 TODO。
3.  **宣告啟動**：
    > 「這是一個輕量級專案，請讀取 `GUIDELINES.md` 並按照 `TASK.md` 開始執行第一項任務。」

### 📂 設定範例 (TASK.md)
```markdown
# 任務：開發「單筆成績修改」功能
- [x] 在 `GradeController` 增加 `Edit` Action (GET/POST)
- [ ] 實作讀取 `STUDENT_GRADE` 原始分數邏輯
- [ ] 實作修改日誌紀錄 (寫入 `SYS_LOG`)
- [ ] 增加「審核狀態」檢核：已鎖定路徑禁止修改
```

## ♻️ 舊專案：如何導入輕量治理

對於現有的舊腳本或混亂的小專案，只需一份「補丁」即可治理：

### SOP 流程：
1.  **快照審計**：由 AI 讀取檔案清單，快速生成現狀描述。
2.  **法規追補**：
    > 「請根據現有代碼，幫我生成一個 20 行以內的 `GUIDELINES.md`，主要規範命名與檔案存放位置即可。」
3.  **任務定錨**：
    > 「我們目前要修正 [某個 Bug]，請幫我建立 `TASK.md` 並列出修復步驟。」

## 📝 輕量級設定範例 (GUIDELINES.md)

以下是一個針對 **ASP.NET MVC 校務系統** 的合併型規範範例：

```markdown
# 校務系統開發規範

## 1. 核心技術棧
*   語言與框架：C# / ASP.NET Core 8.0 MVC
*   資料庫：SQL Server (EF Core / Raw SQL)
*   前端：jQuery, Bootstrap 5, W3.js

## 2. SQL 規範 (MANDATORY)
*   欄位名稱：全大寫並依據 MCP 檢核結果 (如 `STU_ID`)。
*   保留字：全小寫 (如 `select`, `from`, `left join`)。
*   安全：禁止在 SQL 字串中拼接變數，必須使用參數化查詢。

## 3. 命名規則
*   後端 C#：PascalCase (如 `GetGradeList`)。
*   前端 JS：camelCase (如 `gradeDataTable`)。
*   View 名稱：對應 Action 名稱 (如 `Index.cshtml`, `Edit.cshtml`)。

## 4. AI 互動原則
*   每產生一段 SQL，必須說明其對應的校務業務邏輯。
*   完成任務後，務必同步更新 `TASK.md` 的狀態標籤。
```

## 💡 什麼時候該「升級」到專業級？

當您的輕量級專案出現以下徵兆，請考慮參考 [專業級指南](02.動態流程：Conductor 工作流實戰.md)：
1.  檔案數量超過 **20 個**。
2.  開始需要 **多人協作** 或頻繁的 **Git 分支管理**。
3.  出現了大規模的 **Refactoring (重構)** 需求。
4.  需要嚴格的 **TDD (測試驅動開發)** 來確保穩定性。
