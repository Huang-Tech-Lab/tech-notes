# Project Guidelines

本專案的「法律憲法」，定義所有開發人員與 AI Agent 必須遵守的底層規則。

---

## 📅 版本資訊
- **版本**：1.0.0
- **角色**：⚖️ 法律顧問 (Legal Advisor)
- **更新日期**：2026-04-19

---

## 📂 1. 專案記憶與目錄架構

### 1.1 命名規範
- **目錄**：使用 `XX.名稱` (例如 `01.技術筆記`)，確保排序。
- **檔案**：使用全小寫，單字間以 `-` 分割。

### 1.2 記憶保存 (ADR)
- 重大決策必須記錄於 `09.ai/` 相關規範中，不得僅存在於對話歷史。

---

## 📜 2. 代碼治理原則

### 2.1 SQL 規範
- **欄位/表格名稱**：必須使用 `大寫`。
- **保留字**：必須使用 `小寫`。
- **語法**：優先使用 `left join`。

### 2.2 Markdown 規範
- 每個文件必須具備 `Metadata` 區塊（如有需要）。
- 使用 GitHub Alerts (`> [!NOTE]`) 來標註重點。

---

## 🛠️ 3. 執行 SOP
1. 任何更動前，AI 必須檢查本文件。
2. 任何違反本文件的提議需經由法律顧問 (人) 同一認可。
