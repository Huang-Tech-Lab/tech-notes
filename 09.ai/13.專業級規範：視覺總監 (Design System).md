# 13. 專業級規範：視覺總監 (Design System)

本文件詳述專案中的「視覺總監」角色，定義如何透過設計系統規範確保系統 UI/UX 的一致性與高品質感。

---

## 🎨 角色定義：視覺總監 (Design System)

視覺總監的核心目標是提供**「美學秩序」**。他不只管顏色，更管「組件的演進」與「視覺的心理預期」。

### 1. 職責範圍 (Visual Scope)

- **管一致性**：確保所有頁面的按鈕、輸入框、表格樣式統一，避免 UI 碎片化。
- **管美感**：定義調色盤 (Palette) 與間距系統 (Spacing)，提升專案的 premium 感。
- **管互動導引**：定義 Hover、Active、Loading 等狀態的視覺回饋。

### 2. 核心檔案：DESIGN.md

這是專案的「視覺憲法」。AI 在生成任何前端視圖 (HTML/Razor) 前，必須查閱此檔案。

---

## 🛠️ 新專案：如何初始化設計系統

新專案的視覺建立應從「符號」開始：

1.  **定義基礎代幣 (Design Tokens)**：顏色、字體、圓角大小。
2.  **建立原子組件**：定義最基礎的 Button, Input, Table 樣式。
3.  **佈局模型**：定義 Dashboard 與 Form 頁面的標準 Layout。

---

## ♻️ 舊專案：如何進行視覺巡邏 (Audit)

對於混亂的舊系統，視覺總監應執行「去碎片化」：

1.  **搜尋 Inline Styles**：找出所有 `style="..."` 的標記，將其標準化。
2.  **組件抽取**：將重複出現超過 3 次的 HTML 片段（如：學生資訊卡片）抽取為 Razor `PartialView` 或 CSS Class。
3.  **建立規範**：
    > 「請讀取現有的 CSS 檔案，提取出最常使用的 5 種顏色與間距，並將其整理至 `09.ai/DESIGN.md`。」

---

## 📂 範本範例：校務系統設計規範 (DESIGN.md)

以下是針對 **ASP.NET MVC 校務系統** 量身打造且整合具體技術棧的設計範本：

```markdown
# 🏛️ 校務管理系統設計規範 (Design System)

## 1. 核心技術棧 (Technological Foundations)

- **基礎 UI**: Bootstrap (用於佈局、響應式柵格與基礎原子組件)。
- **互動邏輯**: jQuery + jQuery UI (用於進階互動如 Draggable/Sortable)。
- **數據呈現**: DevExtreme (DataGrid, PivotGrid, Charts - 負責複雜報表)。
- **視窗回饋**: SweetAlert2 (取代所有原生 Alert/Confirm)。
- **輕量工具**: w3.js (用於 HTML 自定義屬性包含與簡單數據過濾)。
- **視覺圖表**: Mermaid (用於自動生成業務流程圖與組織架構)。
- **圖示系統**: FontAwesome (僅限免費部分 Free Version，優先使用 Solid 樣式)。

## 2. 視覺基調與顏色

- **主色 (Primary)**: `#1A365D` (學術藍) - 應用於 Bootstrap `.btn-primary` 與 DevExtreme Header。
- **強調色 (Accent)**: `#D69E2E` (學院金) - 用於重要狀態或 FontAwesome 圖示高亮。
- **字體系統**: 繁體中文環境優先使用 `Microsoft JhengHei` (微軟正黑體)。

## 3. 關鍵組件操作規範

### A. 資料表格 (DevExtreme DataGrid)

- **行為**: 必須開啟分頁 (Pager) 與關鍵字搜尋 (Search Panel)。
- **樣式**: 採用雙排標題色，文字對齊遵循：數字右對齊、文字左對齊。
- **按鈕**: 操作列按鈕統一使用 FontAwesome 圖示，並給予 Tooltip 提示。

### B. 彈出視窗 (SweetAlert2)

- **刪除確認**: 強制使用 `warning` 類型，並要求「確認」與「取消」按鈕顏色對比鮮明。
- **成功通知**: 執行成功後使用 `toast` 模式顯示於右上角，不阻斷使用者操作。

### C. 流程展示 (Mermaid)

- **場景**: 每當校務系統涉及「審核流程」時，必須在頁面頂部以 Mermaid 繪製流程圖。
- **風格**: 統一使用 `graph TD` 指令，節點圓角化。

### D. jQuery UI 互動

- **排序功能**: 學生排位或清單手順必須支援 `Draggable` 並在結束後透過 AJAX 同步資料庫。

## 4. 佈局與間距 (Layout System)

- **容器**: 統一使用 Bootstrap `.container-fluid` 以最大化顯示報表數據。
- **標題**: 所有的 H1 下方必須有一條 `border-bottom` 裝飾線，由 jQuery UI 的 `highlight` 類別修飾。

## 5. 視覺心理 (UX Rituals)

- **載入中**: AJAX 請求發起時，統一顯示 DevExtreme 的 `LoadPanel`。
- **空數據**: 使用 FontAwesome 的 `fa-inbox` 圖示搭配文字「目前無任何紀錄」。
```

---

## 🔄 視覺總監的日常維護

1.  **組件庫擴充**：每次新增一種視覺模式，立即記錄到 `DESIGN.md`。
2.  **視覺回歸檢查**：在 Code Review 時，檢查新的 View 是否破壞了既有的 Spacing 規範。
3.  **趨勢升級**：適時將過時的 UI（如過度的漸層）簡化為現代的扁平或微光澤風格。
