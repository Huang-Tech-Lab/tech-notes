# 領域專才：各領域專家 (Specialized Expert) 操作手冊

本文件詳述專案中的「各領域專家」角色，定義其管理的檔案 `CUSTOM_SKILL.md`，以及如何透過專業行為 SOP (Standard Operating Procedure) 提升 AI 在特定領域的執行效率。

---

## 📜 角色定義：各領域專家 (Specialized Expert)

各領域專家的核心目標是提供**「專業直覺」**。他負責管「專業行為」，確保 AI 不僅僅是會寫代碼，而是知道在您的特定環境下「應該怎麼操作」。

### 1. 職責範圍 (Expertise Scope)
*   **管 SOP**：為複雜任務（如部署到生產環境、數據備份、前端動畫）提供 S OP。
*   **管最佳實踐**：提供針對特定庫（如 AWS SDK, Framer Motion）的寫法典範。
*   **管陷阱避雷**：將過去專案中踩過的坑記錄下來，強制 AI 避開。

### 2. 核心檔案：CUSTOM_SKILL.md
這是 AI 的「技能包」，當 AI 讀取到此檔案時，會自動載入特定的行為模式。

---

## 🛠️ 新專案：如何定義領域專家

當您啟動新專案時，應依據「核心技術」定義專家技能：

### SOP 流程：
1.  **識別核心難點**：專案最容易寫錯或最複雜的部分是什麼？（如：複雜的 DB 遷移、AWS Lambda 配置）。
2.  **建立 `CUSTOM_SKILL.md`**：將該難點的標準操作寫入。
3.  **注入範例**：在技能中提供 1-2 個「正確的參考代碼」。

### 範例指令：
> 「根據我們的技術選型 (AWS Amplify)，請為我設計一份 `CUSTOM_SKILL.md`。重點包含：如何正確處理 SSR 環境下的身份驗證，以及如何避免過度調用 API。」

---

## ♻️ 舊專案：如何沉澱專家知識

對於既有專案，專家應從「歷史教訓」中提取知識：

### SOP 流程：
1.  **排查重複修正**：如果某個 Bug 被修了三次，就代表需要建立專家技能。
2.  **提取知識 (Extraction)**：
    > 「請分析最近三個版本的 Bug 修復記錄，並總結出我們在處理『效能瓶頸』時的最佳實踐，將其寫入 `CUSTOM_SKILL.md`。」
3.  **強化約束**：要求 AI 將該 SOP 標註為「開發前必讀」。

## 📂 檔案範例 (CUSTOM_SKILL.md Template)

以下是一個針對 **ASP.NET MVC 網站系統開發** 的領域專家技能範例：

```markdown
# 領域專家技能：ASP.NET MVC 開發與維護 SOP

## 1. 後端架構與 Logic 分流 (Architectural Purity)
當 AI 修改 Controller 時，必須遵循：
1. **瘦 Controller 原則**：Action 內代碼禁止超過 10 行。所有業務邏輯必須遷移至 `Services` 層。
2. **ViewModel 綁定**：嚴禁將 `Models` (DB Entity) 直接傳遞至 View，必須轉換為專用的 `ViewModel`。
3. **依賴注入**：新增功能時，優先檢查是否已有對應的 `Interface` 可進行注入。

## 2. 資料庫操作安全性 (SQL & Performance)
在使用 Dapper 或 Entity Framework 時：
*   **參數化查詢**：絕對禁止使用字串拼接形成 SQL。
*   **非同步優先**：所有 DB 存取必須使用 `async/await` 以避免 Thread Starvation。
*   **效能監控**：查詢超過 3 個 Table 的 Join 語句，必須主動提供執行計畫優化建議。

## 3. 前端交互與 AJAX (UI/UX SOP)
在處理 Razor View 與 JavaScript 時：
*   **PartialView 規範**：重複使用的 UI 區塊（如：學生選單）必須抽取為 `PartialView`。
*   **JSON 回傳格式**：所有 AJAX 回傳必須封裝在 `AjaxResponse<T>` 類別中，包含 `IsSuccess`, `Message`, 與 `Data`。
*   **防呆機制**：在提交 Form 前，必須由專家自動生成 jQuery Validation 檢查語法。

## 4. 代碼陷阱 (MVC Pitfalls)
*   **TempData 誤用**：確保 `TempData` 僅用於跨 Action 的一次性通知。
*   **Session 清理**：禁止將大型 List 存入 Session。
```

---

## 🔄 領域專家的日常維護

1.  **技能迭代**：每當發現新的、更好的組件寫法，立即更新 `CUSTOM_SKILL.md`。
2.  **過期清理**：當庫升級後，舊的「避雷指南」若已無效，應主動刪除以節省 Token。
3.  **Agent 聯動**：品管主管 (TDD) 發現的常見測試失敗，應由領域專家記錄解法。
